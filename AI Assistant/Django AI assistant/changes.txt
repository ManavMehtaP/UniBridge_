The current `schema.prisma` is not sufficient for the full student AI module.

Keep only these schema additions in the main UniBridge project:

1. Chat session metadata for student AI conversations.
Recommended approach:
- add a dedicated `StudentAIChatSession` table linked to `AIConversation`

Required fields:
- `studentId`
- `subjectId` nullable
- `conversationId`
- `title`
- `isDeleted`
- `createdAt`
- `updatedAt`

Reason:
- students need multiple chats, rename chat, and soft delete behavior

2. Study planner persistence.
Required tables:
- `StudyPlan`
- `StudyPlanTask`

Reason:
- current schema has no planner storage, no completion state, and no custom task storage

3. Rich note insight storage.
Required table:
- `NoteInsight`

Reason:
- `Note.aiSummary` alone is not enough for short summary, detailed notes, bullet notes, definitions, formulae, flashcards, and important questions

4. Rich per-file PYQ insight storage.
Required table:
- `PYQInsight`

Reason:
- `PYQAnalysis.topicFrequencies` is aggregate data only and is not enough for detailed per-file extracted AI output

5. Explicit syllabus storage.
Required table:
- `SubjectSyllabus`

Reason:
- the assistant and planner must prioritize syllabus, but the current Prisma schema does not define a syllabus entity

6. Background job persistence.
Required table:
- `StudentAIJob`

Reason:
- note generation, PYQ analysis, study plan generation, and marks retraining need durable job tracking
